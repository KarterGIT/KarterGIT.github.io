The project

A news pipeline that runs while I sleep.

Every day, on a cron schedule, this pipeline pulls the latest posts from tech RSS feeds — NVIDIA and other industry sources — hands them to a locally running Gemma-family model via Ollama for summarization, and posts the digest to a Discord server that pings me. No cloud API, no subscription, no manual step.

The twist: I didn't hand-write it. I directed an AI agent (OpenClaw) end to end — scoping the design, configuring each stage, testing, and correcting — then verified the whole thing ran reliably on its own. The project was as much about learning to operate an agent as about the pipeline itself.

What it taught me

Building it was the easy part

Operating an agent, not just prompting one

OpenClaw wrote the mechanics; I owned the direction — what to build, how the stages fit, what "working" meant, and when its output needed correcting. Directing an agent through a multi-step build is a different skill than chatting with one, and this is where I learned it.

Why a local model

Running Gemma through Ollama means no API bill, no rate limits, and nothing leaving my machine. For an always-on daily job, "free and private" beats "slightly smarter" — and it made the pipeline a live testbed for my local-LLM learning.

The hard part was the plumbing

The model was easy; Discord was the fight — standing up a server, wiring the integration, and getting notifications to actually ping me. Classic automation lesson: the AI is rarely the bottleneck. Connecting systems is.

Verified hands-off

The bar wasn't "it ran once" — it was "it runs when I'm not looking." Scheduled by cron, checked over multiple days, delivering on its own. Reliability, not a demo.

The honest verdict — and what it changed

The pipeline ran flawlessly; the summaries didn't impress me. A small local model produced digests that were serviceable but not worth reading daily — and that was the real lesson: the model is the ceiling of any AI automation. Perfect plumbing can't fix a weak brain. That conclusion is part of why I acquired an NVIDIA DGX Spark — to run bigger, better local models and raise that ceiling. The next iteration of this pipeline gets a stronger engine.

Why it matters

Shipping something real teaches what tutorials can't.

This project covered the full arc: direct an agent to build a system, wire it into real tools, verify it runs unattended, judge the output honestly — and let that judgment drive the next investment. That loop is how I approach every AI project since.
