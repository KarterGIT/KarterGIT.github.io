Computer Science Fundamentals CSCI 212 · Semester Coursework
The project

Building the machinery under modern software — by hand.

Over a full semester I implemented the core structures that everything in computing sits on — linked lists, stacks, queues, binary search trees, heaps, and hash tables — from scratch, without leaning on built-in library versions. The point wasn't the language; it was understanding how the machinery works: how data is organized, traversed, and moved efficiently.

The same thinking shows up everywhere in my AI work — pipelines, queues, and search are the bones of every system I touch. Implemented in Java across 90+ source files of weekly labs and projects.

What I built

Seven fundamentals, from first principles

Linked structures & generics

Wrote a generic singly linked list (nodes, traversal, insertion/removal) and a custom ArrayList behind my own List interface — the "how do collections actually work" layer most people never open up.

Stacks & queues

Built a linked-list stack and queue from scratch, then put them to work — processing job-style workloads (sounds, images) in order. The same pattern behind schedulers, undo systems, and message pipelines.

Sorting — with the stopwatch out

Implemented bubble and insertion sort in ascending/descending variants over object data, then benchmarked them in nanoseconds with a custom timer to compare algorithms with real measurements — not just Big-O on paper.

Trees & heaps

Implemented a binary search tree with traversals and a generic heap with pluggable comparators — then used it to drive heap sort. Priority ordering, the structure underneath schedulers and leaderboards.

Hashing & search

Built hash tables and implemented linear and binary search, comparing how data layout changes what "fast lookup" even means — the trade-off at the heart of every database and cache.

Recursion

Solved problems recursively and iteratively side-by-side (power functions, category-tree search) to understand when self-reference simplifies a problem — and when it quietly costs you a call stack.

Object-oriented design, GUIs & file I/O

Tied it together with real applications: interface-driven, polymorphic designs (a pay-calculation system, a clinic billing model) and desktop GUI apps with input validation and file persistence — structure, behavior, and data meeting in one working program.

Why it matters

Fundamentals are the part that transfers.

Languages come and go — the structures don't. Queues show up in my AI automation pipelines, trees and search in how models organize data, and benchmarking in how I verify that systems actually run well. This course is where that foundation got built.
